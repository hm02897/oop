Object Oriented Programming & Design Methodology

The concept of Object Oriented Programming & Design is to think in terms of individual objects as opposed to functions that manipulate data. You distribute a problem into a series of objects. Every object holds in itself all the functionality related to that object. Mostly, the internal data is hidden so that the user does not need to worry about how the data is used as long as the object gives a desired response.

This makes the code easier to manage and debug because we can isolate the problem to an object rather than finding the function that is causing error. Another useful feature is the code reusability and inheritance. Once a suitable code has been made, we can use it in any future project without the need of rewriting the code. At the same time, we can create new code that builds on the previous one by inheriting its properties and functionalities.

The aim of this course is to help students understand the Object Oriented Programming & Design through intensive programming activities. Students do this by doing weekly assignments where they learn the basics for creating a game. By doing so, they learn about memory management, real-time object interaction and data structures. With visual feedback, the students get a good idea about the underlying memory management issues and acquire problem solving capabilities.
